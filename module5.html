Модуль 5: Работа с BIOS и образами — пути к ПЗУ, загрузка ОС

✍️ Время чтения: ~20 минут | 🎯 Цель: научиться правильно указывать пути к ПЗУ, монтировать образы дисков и понимать процесс загрузки операционной системы в PCx86

В предыдущих модулях мы собрали виртуальную машину и подключили все устройства. Однако без программного обеспечения — BIOS и операционной системы — наш эмулятор остаётся просто «железом». В этом модуле мы разберёмся, как работают образы ПЗУ (ROM) и дисков (FDD/HDD), какие форматы поддерживает PCx86, и как правильно настроить загрузку операционной системы.

💡 Ключевая идея
Эмулятор PCx86 не работает с обычными бинарными файлами (BIN/IMG) напрямую — ему требуются специальные JSON-образы, которые содержат не только данные, но и служебную информацию (геометрию диска, CRC и т.д.). Однако вы можете использовать как готовые JSON-образы с сайта PCjs, так и конвертировать собственные IMG-образы с помощью утилиты DiskDump.

5.1. Образы ПЗУ (ROM) — BIOS и видеоПЗУ

Компонент <rom> в PCx86 отвечает за загрузку образов постоянного запоминающего устройства. Чаще всего это системный BIOS (адрес 0xF0000) и видеоПЗУ (адрес 0xC0000).

5.1.1. Атрибуты тега <rom>

  • addr — физический адрес, по которому ПЗУ отображается в памяти (обязательный).
  • size — размер ПЗУ в байтах (обычно 0x10000 = 64 КБ для системного BIOS).
  • alias — альтернативный адрес (например, 0xFF0000 для совместимости с сегментацией).
  • file — путь к JSON-файлу с образом ПЗУ (обязательный).
  • notify — идентификатор компонента, который должен быть уведомлён о загрузке ПЗУ (используется для видеокарт).

Пример подключения системного BIOS для IBM PC AT:

<rom id="romBIOS" addr="0xf0000" size="0x10000" alias="0xff0000" file="/machines/pcx86/ibm/5170/rom/bios/1985-11-15/ATBIOS-REV3.json5" />

5.1.2. Формат JSON5 для образов ПЗУ

PCjs использует формат JSON5 (расширение .json5) для хранения образов ПЗУ. JSON5 — это надмножество JSON, позволяющее использовать комментарии, необязательные кавычки и другие удобства для ручного редактирования[reference:0]. Структура файла представляет собой массив 32-битных значений (обычно в шестнадцатеричном формате), которые соответствуют байтам ПЗУ. Например:

[
    0xEA, 0x5B, 0xE0, 0x00, 0xF0,  // JMP &F000:E05B
    0x31, 0xC0, 0x8E, 0xD0, 0xBC,  // XOR AX,AX; MOV SS,AX; MOV SP,...
    ...
]

Вы можете скачать готовые образы BIOS с сайта PCjs или создать свои с помощью утилиты FileImage.js, которая конвертирует бинарные файлы в JSON5[reference:1].

⚠️ Важно: Без правильно подключённого образа BIOS машина не запустится. Убедитесь, что путь к файлу указан верно и сам файл не повреждён. Если BIOS отсутствует, в консоли браузера появится ошибка 404.

5.2. Загрузка операционной системы (boot process)

Процесс загрузки в IBM PC-совместимых компьютерах стандартизирован:

  1. После включения питания процессор начинает выполнение кода по адресу 0xFFFF0 (в ПЗУ BIOS).
  2. BIOS выполняет POST (Power-On Self Test), инициализирует оборудование и ищет загрузочное устройство (дискету, жёсткий диск).
  3. Загрузчик BIOS считывает первый сектор (boot sector) с устройства и передаёт ему управление.
  4. Boot sector загружает в память основную часть операционной системы (например, MS-DOS или OS/2).

В PCx86 этот процесс полностью эмулируется. Всё, что нужно сделать разработчику — подключить образ диска с правильной загрузочной записью.

5.3. Образы гибких дисков (FDD)

PCx86 поддерживает образы дискет в двух форматах: JSON (рекомендуемый) и IMG (бинарный). Однако для работы с IMG-образами требуется дополнительная настройка, поэтому мы сосредоточимся на JSON.

5.3.1. Формат JSON для образов дискет

Согласно документации PCjs, каждый JSON-образ дискеты представляет собой массив объектов Cylinder, где каждый Cylinder — это массив Track (по одному на каждую головку), а каждый Track — массив Sector. Каждый сектор содержит минимальный набор свойств[reference:2]:

{
    "sector": 1,        // идентификатор сектора (1-18 для 1.44 МБ)
    "length": 512,      // размер сектора в байтах (обычно 512)
    "data": [0xEB, ...] // массив байтов данных (или 32-битных значений)
}

Такой формат позволяет эмулировать нестандартные дискеты (например, с секторами не по 512 байт или с ошибками CRC), что необходимо для работы с защищённым от копирования ПО[reference:3].

5.3.2. Подключение образа дискеты в XML

Для автоматической загрузки дискеты при старте используйте атрибут autoMount компонента <fdc>:

<fdc autoMount="{A:{name:'MS-DOS 6.22', path:'msdos622.json'}}" />

Если нужно предоставить пользователю выбор из нескольких образов, используйте тег <disk> внутри <listDisks>:

<fdc>
    <control type="list" binding="listDisks">
        <disk id="MSDOS622" name="MS-DOS 6.22" />
        <disk id="UTILS" name="Утилиты" />
    </control>
</fdc>

5.4. Образы жёстких дисков (HDD)

Жёсткие диски в PCx86 также используют JSON-формат, но с более сложной структурой. Образ содержит не только секторы, но и параметры геометрии (цилиндры, головки, секторы на дорожку).

5.4.1. Формат JSON для жёстких дисков

Типичный JSON-образ жёсткого диска включает поле "disk" с описанием геометрии и поле "tracks" с массивом дорожек. Пример из репозитория PCjs:

{
    "disk": {
        "cylinders": 306,
        "heads": 4,
        "sectors": 17,
        "model": "ST-225"
    },
    "tracks": [ ... ] // массив данных секторов
}

Вы можете создавать такие образы с помощью утилиты DiskDump, которая конвертирует бинарные IMG-образы в JSON[reference:4].

5.4.2. Подключение жёсткого диска в XML

Используйте компонент <hdc> с атрибутом drives:

<hdc id="hdcAT" type="AT" drives="[{name:'20Mb Hard Disk', type:2, path:'hdd20mb.json'}]" />

Параметр type определяет геометрию диска согласно таблице типов в PCjs. Для PC AT тип 2 соответствует 20-мегабайтному диску (615 цилиндров, 4 головки, 17 секторов).

5.5. Конвертация бинарных образов в JSON

Если у вас есть собственный IMG-образ дискеты или жёсткого диска, его можно конвертировать в JSON с помощью утилиты DiskDump (входит в состав PCjs)[reference:5].

Пример команды для конвертации IMG-образа дискеты:

node modules/diskdump/bin/diskdump --disk=myimage.img --format=json --output=myimage.json

Для жёсткого диска:

node modules/diskdump/bin/diskdump --disk=hdd.img --format=json --output=hdd.json

Также DiskDump может создавать образы из папок с файлами, что удобно для сборки собственных загрузочных дискет[reference:6].

📌 Примечание: Утилита DiskDump требует установленной платформы Node.js. Если вы не хотите устанавливать Node.js, вы можете использовать онлайн-версию PCjs для конвертации: загрузите IMG-образ в эмулятор и нажмите кнопку Save — файл сохранится в JSON-формате.

5.6. Практические примеры

Пример 1: Загрузка MS-DOS 6.22 с дискеты

<machine id="my-pc" type="pcx86">
    <name>IBM PC with MS-DOS 6.22</name>
    <computer id="pc" name="IBM PC" buswidth="20" />
    <cpu model="8088" cycles="4770000" autostart="true" />
    <ram addr="0x00000" size="0xa0000" />
    <rom addr="0xf0000" size="0x10000" alias="0xff0000" file="pcbios.json5" />
    <video model="cga" memory="0x4000" screenWidth="640" screenHeight="400" />
    <fdc autoMount="{A:{name:'MS-DOS 6.22', path:'msdos622.json'}}" />
    <keyboard model="us84" />
</machine>

При запуске эмулятор загрузится с дискеты A: и вы увидите приглашение MS-DOS.

Пример 2: Загрузка OS/2 1.0 с жёсткого диска

<machine id="os2-machine" type="pcx86">
    <name>IBM PC AT with OS/2 1.0</name>
    <computer id="at" name="IBM PC AT" buswidth="24" />
    <cpu model="80286" cycles="8000000" autostart="true" />
    <ram addr="0x00000" size="0xa0000" />
    <ram addr="0x100000" size="0x160000" />
    <rom addr="0xf0000" size="0x10000" alias="0xff0000" file="atbios.json5" />
    <video model="ega" memory="0x20000" screenWidth="1280" screenHeight="700">
        <rom addr="0xc0000" size="0x4000" file="egabios.json5" notify="videoEGA" />
    </video>
    <hdc type="AT" drives="[{name:'OS/2 1.0', type:2, path:'os2hdd.json'}]" />
    <keyboard model="us84" />
</machine>
📝 Задание для самопроверки
1. Скачайте образ системного BIOS для IBM PC 5150 (например, PCBIOS-REV1.json5) с сайта PCjs и подключите его в своей конфигурации.
2. Создайте образ дискеты с программой на BASIC с помощью DiskDump (или используйте готовый образ) и настройте автозагрузку.
3. Настройте эмулятор PC AT так, чтобы он загружался с жёсткого диска (используйте образ OS/2 1.0 с сайта PCjs).
4. Попробуйте сконвертировать свой IMG-образ в JSON с помощью DiskDump и загрузить его в эмулятор.

5.7. Устранение неполадок

  • Ошибка 404 при загрузке BIOS — проверьте путь к файлу. Если файл лежит в подпапке, укажите полный относительный путь (например, file="rom/atbios.json5").
  • Non-system disk or disk error — образ дискеты не загрузочный. Убедитесь, что дискета содержит загрузочный сектор (например, образ MS-DOS).
  • Ошибка "Invalid JSON" — файл образа повреждён или имеет неверную структуру. Попробуйте переконвертировать его с помощью DiskDump.
  • Эмулятор не видит жёсткий диск — проверьте атрибуты type и path. Для PC AT обязательно укажите type="AT" в теге <hdc>.
