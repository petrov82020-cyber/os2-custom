Глава 2: Создание HTML-страницы и загрузка на GitHub

✍️ Время чтения: ~15 минут | 🎯 Цель: создать первую веб-страницу и опубликовать её через GitHub Pages

В первой главе мы зарегистрировались на GitHub, создали репозиторий и включили GitHub Pages. Но сайт пока не работает – у нас нет главной страницы index.html. В этой главе мы создадим её, добавим стили и загрузим в репозиторий. После этого ваш сайт станет доступен в интернете!

💡 Что такое HTML и CSS?
HTML (HyperText Markup Language) – язык разметки, который описывает структуру страницы (заголовки, абзацы, картинки). CSS (Cascading Style Sheets) – язык стилей, который отвечает за внешний вид: цвета, шрифты, расположение элементов.

2.1 Создание файла index.html

Откройте любой текстовый редактор (Блокнот, Notepad++, VS Code). Создайте новый файл и назовите его index.html. Внутрь вставьте следующий код:

<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Мой первый сайт</title>
</head>
<body>
    <h1>Привет, мир!</h1>
    <p>Это мой первый сайт, размещённый на GitHub Pages.</p>
</body>
</html>

Сохраните файл. Это базовая страница, которая будет отображаться при открытии вашего сайта.

2.2 Добавление стилей (CSS)

Чтобы страница выглядела привлекательнее, добавим немного стилей. Внутри тега <head> добавьте блок <style>:

<style>
    body {
        font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
        max-width: 800px;
        margin: 0 auto;
        padding: 20px;
        background: #f5f5f5;
        color: #333;
    }
    h1 {
        color: #0366d6;
        border-bottom: 2px solid #0366d6;
        padding-bottom: 10px;
    }
</style>

Теперь ваша страница будет иметь аккуратный вид: ограниченная ширина, красивый шрифт, синий заголовок с подчёркиванием.

2.3 Проверка локально

Дважды кликните по файлу index.html – он откроется в вашем браузере. Вы увидите результат. Так вы можете тестировать любые изменения, не загружая их на GitHub. Это очень удобно.

⚠️ Важно: Имя главной страницы должно быть именно index.html (все буквы маленькие, без опечаток). GitHub Pages автоматически ищет этот файл. Если назвать его по-другому, сайт не откроется.

2.4 Загрузка файла в репозиторий (через веб-интерфейс)

Самый простой способ загрузить файл – через сайт GitHub.

  1. Перейдите в ваш репозиторий: https://github.com/ваше_имя/my-first-site.
  2. Нажмите на кнопку Add file → Upload files.
  3. Перетащите файл index.html из папки компьютера в область загрузки.
  4. Внизу страницы напишите комментарий, например «Добавлена главная страница».
  5. Нажмите Commit changes.

Через 1-2 минуты GitHub Pages обновит ваш сайт. Перейдите по адресу https://ваше_имя.github.io/my-first-site/ – вы должны увидеть свою страницу!

🎉 Поздравляю! Ваш первый сайт опубликован в интернете. Теперь он доступен любому человеку, у кого есть ссылка.

2.5 Редактирование файла на GitHub (без скачивания)

Вы можете изменять index.html прямо на сайте, не используя локальный редактор.

  1. В репозитории откройте файл index.html.
  2. Нажмите на значок карандаша (Edit).
  3. Внесите изменения, например, добавьте ещё один абзац: <p>Я учусь создавать сайты!</p>.
  4. Напишите комментарий к изменению и нажмите Commit changes.

Сайт обновится автоматически через минуту. Это отличный способ быстро исправлять ошибки.

2.6 Что такое коммит и почему он важен

Коммит (commit) – это сохранение изменений с комментарием. GitHub хранит историю всех коммитов, поэтому вы всегда можете вернуться к предыдущей версии файла. Это как «точка сохранения» в игре.

Чтобы посмотреть историю, откройте файл index.html и нажмите на History (кнопка с часами). Вы увидите все версии с датами и комментариями.

📝 Задание для самопроверки
1. Создайте на компьютере файл index.html с базовой структурой и стилями.
2. Добавьте на страницу своё имя и список ваших любимых книг или фильмов (используйте теги <ul> или <ol>).
3. Загрузите файл в репозиторий через веб-интерфейс.
4. Дождитесь обновления и откройте свой сайт. Убедитесь, что изменения отобразились.
5. Внесите ещё одно изменение (например, измените цвет фона) прямо через редактор GitHub и снова проверьте.

2.7 Что делать, если сайт не обновляется?

  • Подождите ещё 2-3 минуты – иногда GitHub Pages требуется немного времени.
  • Убедитесь, что файл называется index.html (а не Index.html или index.html.txt).
  • Проверьте, что в настройках Pages выбрана правильная ветка (main или master) и папка / (root).
  • Откройте страницу в режиме инкогнито или очистите кэш браузера (Ctrl+Shift+Delete).
