Глава 3: Основы Git и командной строки. Добавление изображений и CSS

✍️ Время чтения: ~20 минут | 🎯 Цель: освоить Git через командную строку, добавить изображения и подключить внешний CSS

В предыдущих главах мы загружали файлы через веб-интерфейс GitHub. Но что делать, если нужно загрузить много файлов или файл больше 25 МБ (как в нашем эксперименте с эмулятором)? Тогда пригодится Git – система контроля версий, которая работает через командную строку. В этой главе мы освоим базовые команды Git, а также научимся добавлять на сайт изображения и выносить стили в отдельный файл.

💡 Зачем нужна командная строка?
Командная строка (терминал, Git Bash) позволяет выполнять действия, которые недоступны через веб-интерфейс: загружать файлы любого размера, автоматизировать процессы, работать с ветками и историей изменений.

3.1 Установка Git (если ещё не установлен)

Для Windows 7 нужно установить последнюю поддерживаемую версию – Git 2.46.2. Скачайте 64-битную версию с официального сайта или напрямую:

https://github.com/git-for-windows/git/releases/download/v2.46.2.windows.1/Git-2.46.2-64-bit.exe

При установке оставляйте параметры по умолчанию, но убедитесь, что отмечена опция Git from the command line and also from 3rd-party software (это позволит использовать Git из обычной командной строки). После установки перезагрузите компьютер.

3.2 Первоначальная настройка Git

Откройте Git Bash (или командную строку) и выполните две команды, чтобы Git знал, кто вы:

git config --global user.name "Ваше Имя"
git config --global user.email "ваш_email@example.com"

Имя и email могут быть любыми, но лучше использовать те же, что и в аккаунте GitHub – для удобства.

3.3 Клонирование репозитория

Клонировать – значит скачать копию репозитория с GitHub на ваш компьютер. Это нужно, чтобы работать с файлами локально.

  1. На странице вашего репозитория нажмите зелёную кнопку Code, скопируйте HTTPS-ссылку (например, https://github.com/ваше_имя/my-first-site.git).
  2. В Git Bash перейдите в папку, где хотите хранить проекты (например, cd Desktop).
  3. Выполните: 
    git clone https://github.com/ваше_имя/my-first-site.git
  4. Перейдите в созданную папку: cd my-first-site.

Теперь у вас есть локальная копия репозитория. Вы можете открыть её в Проводнике – там лежат все файлы, включая index.html.

⚠️ Важно: Если вы уже загружали файлы через веб-интерфейс, при клонировании они скачаются автоматически. Если репозиторий пуст – вы получите пустую папку (кроме скрытой папки .git).

3.4 Основные команды Git: add, commit, push, pull

Рассмотрим самый частый сценарий: вы добавили или изменили файлы локально и хотите отправить изменения на GitHub.

  • git add . – добавить все изменённые файлы в «зону подготовки» (staging area). Можно указать конкретный файл: git add index.html.
  • git commit -m "сообщение" – зафиксировать изменения с комментарием (например, git commit -m "Добавил картинку").
  • git push origin main – отправить коммиты на GitHub (в ветку main).
  • git pull origin main – забрать свежие изменения с GitHub (если кто-то работал удалённо).

Попробуем на практике. Создайте в локальной папке репозитория новый файл, например style.css, и напишите в нём несколько CSS-правил. Затем выполните:

git add .
git commit -m "Добавлен файл стилей"
git push origin main

После этого файл появится на GitHub.

📌 Важное замечание: Перед первым push на GitHub может потребоваться авторизация. Если Git запрашивает пароль, нужно ввести личный токен доступа (PAT) вместо пароля. Как создать токен, описано в предыдущих главах (Settings → Developer settings → Personal access tokens).

3.5 Добавление изображений на сайт

Чтобы украсить страницу, добавим картинку. Скачайте любое изображение (например, логотип GitHub) и положите его в папку репозитория. Назовём файл github-logo.png.

Теперь отредактируйте index.html, добавив тег <img>:

<img src="github-logo.png" alt="Логотип GitHub" width="150">

Атрибут src – путь к файлу (если изображение лежит в той же папке, что и HTML, достаточно имени). alt – текст, который отображается, если картинка не загрузилась.

После добавления картинки выполните:

git add .
git commit -m "Добавлено изображение"
git push origin main

Через минуту картинка появится на вашем сайте.

3.6 Подключение внешнего CSS-файла

До сих пор мы писали стили прямо в HTML (в теге <style>). Для больших проектов лучше выносить CSS в отдельный файл – это упрощает поддержку.

  1. Создайте файл style.css в той же папке, что и index.html.
  2. Перенесите все стили из <style> в style.css (без тегов, просто CSS-правила).
  3. В index.html в блоке <head> удалите блок <style> и добавьте ссылку на внешний файл: 
    <link rel="stylesheet" href="style.css">

Пример содержимого style.css:

body {
    font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
    max-width: 800px;
    margin: 0 auto;
    padding: 20px;
    background: #f5f5f5;
    color: #333;
}
h1 {
    color: #0366d6;
    border-bottom: 2px solid #0366d6;
    padding-bottom: 10px;
}
img {
    border-radius: 12px;
    box-shadow: 0 4px 8px rgba(0,0,0,0.1);
}

После этого сохраните изменения, сделайте коммит и push. Сайт должен остаться таким же красивым, но теперь код организован лучше.

📝 Задание для самопроверки
1. Установите Git (если ещё не установлен) и настройте имя/email.
2. Клонируйте ваш репозиторий my-first-site на компьютер.
3. Добавьте в папку любое изображение (например, ваше фото или картинку из интернета).
4. Измените index.html – добавьте картинку и, возможно, подпись.
5. Создайте внешний файл style.css и перенесите в него все стили.
6. Выполните git add ., git commit -m "Обновление: картинка и внешний CSS", git push origin main.
7. Проверьте, что сайт обновился и всё работает.

3.7 Что делать, если Git не видит файлы или выходит ошибка?

  • fatal: not a git repository – вы находитесь не в той папке. Убедитесь, что выполнили cd my-first-site.
  • rejected (non-fast-forward) – удалённый репозиторий содержит изменения, которых нет у вас. Сделайте git pull origin main, затем снова git push.
  • error: src refspec main does not match any – возможно, ваша ветка называется master, а не main. Проверьте командой git branch и замените в push main на master.
  • LF will be replaced by CRLF – предупреждение о переносах строк, его можно игнорировать.
