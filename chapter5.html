5.1 Добавление формы обратной связи (без бэкенда)

Статические сайты не могут обрабатывать формы самостоятельно, но существуют сторонние сервисы, которые принимают данные и пересылают их на email. Один из самых простых – Formspree.

Как подключить Formspree

  1. Зарегистрируйтесь на formspree.io (бесплатный тариф – 50 отправок в месяц).
  2. Создайте новую форму – получите endpoint типа https://formspree.io/f/ваш_токен.
  3. Добавьте на страницу HTML-форму:
<form action="https://formspree.io/f/ваш_токен" method="POST">
    <label>Ваше имя:</label>
    <input type="text" name="name" required>
    <label>Email:</label>
    <input type="email" name="_replyto" required>
    <label>Сообщение:</label>
    <textarea name="message" rows="5" required></textarea>
    <button type="submit">Отправить</button>
</form>

После отправки данные появятся в панели Formspree, а также могут дублироваться на ваш email. Это полностью бесплатно для небольших проектов.

⚠️ Примечание: Formspree ограничивает количество бесплатных отправок. Если ожидаете много писем, рассмотрите альтернативы: Netlify Forms, Getform, или напишите простой сервер на JavaScript (но тогда нужен будет хостинг с бэкендом).

5.2 Основы SEO (поисковой оптимизации)

Чтобы ваш сайт находили в Google, следуйте простым правилам:

  • Заголовок страницы – тег <title> должен быть уникальным и описывать содержание.
  • Мета-описание – добавьте <meta name="description" content="Краткое описание"> в <head>.
  • ЧПУ-ссылки – используйте понятные URL (например, /about вместо /page?id=123). В статике это просто имена файлов.
  • Альтернативный текст для изображений – атрибут alt помогает поисковым роботам понять, что на картинке.
  • Адаптивный дизайн – сайт должен хорошо выглядеть на мобильных устройствах (используйте метатег viewport и медиазапросы).
  • Скорость загрузки – оптимизируйте изображения (сжимайте через TinyPNG), не подключайте тяжёлые скрипты.

Для проверки индексации добавьте свой сайт в Google Search Console и отправьте sitemap.xml (Jekyll может генерировать его автоматически с плагином).

5.3 Как продвигать сайт: социальные сети, обмен ссылками

  • Разместите ссылку на сайт в профилях соцсетей (Twitter, LinkedIn, GitHub).
  • Участвуйте в тематических сообществах (Reddit, Telegram, форумы) – делитесь полезными материалами, не спамьте.
  • Попросите друзей или коллег разместить ссылку на ваш сайт на их ресурсах.
  • Создайте качественный контент – статьи, видеоуроки, инфографику – чтобы ссылались сами.

5.4 Частые ошибки и их решение

  • Сайт не обновляется после push: подождите 2 минуты, очистите кэш браузера (Ctrl+Shift+Delete).
  • 404 Not Found: проверьте, что файл называется index.html и лежит в корне выбранной ветки.
  • CSS не применяется: убедитесь, что путь к файлу указан верно (например, href="style.css"), и файл загружен в репозиторий.
  • Jekyll игнорирует файлы: если вы не используете Jekyll, создайте пустой файл .nojekyll в корне.
  • Картинки не отображаются: проверьте регистр символов в имени файла (на Linux-серверах GitHub различается регистр).
  • Git push rejected: сделайте git pull origin main перед push, разрешите конфликты.

  • MDN Web Docs – полный справочник по HTML, CSS, JavaScript.
  • Официальная документация GitHub Pages – все возможности платформы.
  • Pro Git (книга) – углублённое изучение Git.
  • Jekyll Documentation – создание блогов и сайтов на Markdown.
  • GitHub Actions Documentation – автоматизация сборки, тестирования, деплоя.
  • HTML Academy – интерактивные курсы по вёрстке (русский язык).
  • Figma – бесплатный инструмент для дизайна сайтов.
📝 Финальное задание
1. Добавьте на ваш сайт форму обратной связи через Formspree (или другой сервис).
2. Проверьте мета-теги и заголовки на всех страницах.
3. Создайте карту сайта (sitemap.xml) – в Jekyll это делается плагином jekyll-sitemap, добавьте его в _config.yml: plugins: [jekyll-sitemap].
4. Отправьте sitemap в Google Search Console.
5. Опубликуйте ссылку на ваш сайт в соцсетях с хештегом #мойпервыйсайт.
