📘 Как опубликовать сайт на GitHub Pages

Полное видеоруководство для начинающих (2026) от канала The Common Coder

🎥 О чём видео: Вы хотите разместить свой проект на GitHub Pages, но не знаете, с чего начать? В этом руководстве мы шаг за шагом разберём всё, что нужно знать, чтобы начать работать с GitHub Pages и хостить свои проекты уже сегодня. (Автор: Джош, канал The Common Coder)

1. Что такое GitHub Pages?

GitHub Pages – это бесплатный сервис хостинга статических сайтов, предоставляемый GitHub. Он идеально подходит для документации, портфолио, лендингов и фронтенд-проектов.

📌 Ключевые особенности:
  • ❌ Нет серверной обработки (PHP, Node.js, Ruby on Rails) – только статические файлы.
  • ✅ HTML, CSS, JavaScript работают в полную силу.
  • ✅ Можно подключаться к внешним API через JavaScript.
  • ✅ Поддержка фреймворков (React, Vue, Angular) – если вы закоммитите результат сборки или настроите GitHub Actions.

2. Типы сайтов: пользовательский vs проектный

👤 User site (пользовательский)

Один на аккаунт. Адрес: https://ваше_имя.github.io
Представляет вас как разработчика.

📁 Project site (проектный)

Неограниченное количество. Адрес: https://ваше_имя.github.io/название_репозитория
Создаётся для каждого репозитория.

3. Настройка репозитория для GitHub Pages

  1. Перейдите в Settings вашего репозитория.
  2. В левом меню выберите Pages.
  3. В разделе Branch выберите ветку (например, main или master).
  4. Укажите папку: обычно / (root) (если ваш index.html лежит в корне).
  5. Нажмите Save.

💡 Если вы уже публиковали сайт ранее, и ветка уже выбрана – сначала переключите её на None, сохраните, а затем снова выберите нужную ветку. Это принудительно запустит сборку.

4. Доступ к сайту и отслеживание сборки

После сохранения GitHub автоматически запускает процесс развёртывания. Перейдите во вкладку Actions вашего репозитория – вы увидите workflow pages build and deployment. Когда появится зелёная галочка ✅, сайт готов.

🌐 Адрес сайта: https://ваше_имя.github.io/название_репозитория
Ссылка появится в настройках Pages после успешной сборки.

5. Обновление и автоматическое переразвёртывание

Одно из главных преимуществ – автоматическое обновление сайта при каждом изменении в выбранной ветке.

  • Вносите изменения локально → git commit → git push.
  • GitHub сам запускает новую сборку и через 1–2 минуты сайт обновляется.

Никаких дополнительных действий не требуется!

6. Дополнительные возможности

  • Пользовательский домен: в настройках Pages есть поле Custom domain – можно привязать свой домен (например, my-site.com).
  • Приватные репозитории: GitHub Pages может публиковать сайты и из приватных репозиториев (для бесплатных аккаунтов обычно работает, но проверьте актуальные условия).
