Модуль 3: Создание минимальной конфигурации XML с нуля

✍️ Время чтения: ~20 минут | 🎯 Цель: научиться писать собственные XML-файлы для эмулятора PCx86

В предыдущем модуле мы использовали готовую конфигурацию, полученную через Save Machine. Теперь пришло время создать свою собственную виртуальную машину «с нуля» – написать XML-файл, в котором вручную опишем процессор, память, видеокарту, диски и другие компоненты. Это даст нам полный контроль и понимание того, как работает эмулятор.

💡 Что такое XML-конфигурация?
XML-файл – это текстовый документ, в котором теги описывают все устройства виртуальной машины. Эмулятор PCx86 читает этот файл и создаёт соответствующие JavaScript-объекты. Вы можете редактировать XML в любом текстовом редакторе (Блокнот, VS Code).

3.1. Структура минимального XML

Любая конфигурация начинается с корневого тега <machine> и содержит дочерние теги для каждого компонента. Вот минимальный набор, достаточный для загрузки IBM PC 5150:

<machine id="my-pc" type="pcx86">
    <name>Мой первый эмулятор</name>
    <computer id="my-computer" name="IBM PC" buswidth="20" />
    <cpu model="8088" cycles="4770000" autostart="true" />
    <ram addr="0x00000" size="0xa0000" />
    <rom addr="0xf0000" size="0x10000" alias="0xff0000" file="bios.json5" />
    <video model="cga" memory="0x4000" screenWidth="640" screenHeight="400" />
    <fdc />
    <keyboard />
</machine>

Этот файл описывает компьютер с процессором 8088 (4.77 МГц), 640 КБ ОЗУ, CGA-видеокартой и подключённым ПЗУ BIOS. Эмулятор запустится, но без образов дисков вы увидите только экран BASIC или сообщение об ошибке.

3.2. Подробный разбор тегов

<machine>

Корневой элемент. Обязательные атрибуты: id (уникальный идентификатор) и type (всегда "pcx86"). Также можно задать внешний вид: border, pos, background.

<name>

Отображаемое название машины (появляется в интерфейсе). Необязательно, но полезно.

<computer>

Определяет модель системной платы. Атрибуты: id, name (любое имя), buswidth (20 для XT, 24 для AT).

<cpu>

Процессор. Основные атрибуты:

  • model – тип: "8088", "80186", "80286", "80386".
  • cycles – тактовая частота в герцах (например, 4770000 для 4.77 МГц).
  • autostart – "true" или "false" (запускать ли выполнение кода сразу).

<ram>

Оперативная память. Атрибуты:

  • addr – начальный физический адрес (обычно 0x00000).
  • size – размер в байтах (шестнадцатеричный). Для 640 КБ: 0xa0000.
  • test – "false" отключает тест памяти в BIOS (ускоряет загрузку).
Можно добавить несколько блоков RAM, например для расширенной памяти (адрес 0x100000).

<rom>

ПЗУ (обычно BIOS). Атрибуты:

  • addr – адрес отображения (обычно 0xf0000 для BIOS).
  • size – размер ПЗУ (обычно 0x10000 = 64 КБ).
  • alias – альтернативный адрес (например, 0xff0000 для совместимости).
  • file – путь к JSON-файлу с образом ПЗУ.

<video>

Видеоадаптер. Атрибуты:

  • model – "mda", "cga", "ega", "vga".
  • memory – объём видеопамяти (для CGA достаточно 0x4000 = 16 КБ).
  • screenWidth, screenHeight – размер canvas в пикселях (рекомендуется пропорционально исходному разрешению).
Для EGA/VGA также нужно добавить вложенный тег <rom> для ПЗУ видеокарты (адрес 0xc0000).

<fdc> и <hdc>

Контроллеры дисков. Для FDC можно указать autoMount с образами дискет. Для HDC – атрибут drives с массивом жёстких дисков. Пример подключения образа дискеты:

<fdc autoMount="{A:{name:'MS-DOS 6.22', path:'msdos622.img'}}" />

Для жёсткого диска:

<hdc drives="[{name:'20Mb HDD', type:2, path:'myhdd.img'}]" />

<keyboard>, <mouse>, <panel>, <chipset>

Эти теги необязательны, но добавляют удобство. <keyboard> можно указать раскладку (model="us84"). <mouse serial="com1" /> подключает мышь. <chipset model="5170" /> задаёт чипсет для AT-совместимых машин.

3.3. Создание работающей конфигурации (пример)

Давайте создадим XML для эмуляции IBM PC 5150 с 640 КБ RAM, CGA-видео и возможностью загружать MS-DOS с дискеты. Предположим, у нас есть файлы: ibm5150-bios.json5 (ПЗУ), msdos622.img (образ дискеты).

<machine id="my-ibm5150" type="pcx86" border="1" pos="center">
    <name>IBM PC 5150 (MS-DOS 6.22)</name>
    <computer id="pc5150" name="IBM PC" buswidth="20" />
    <cpu model="8088" cycles="4770000" autostart="true" />
    <ram addr="0x00000" size="0xa0000" />
    <rom addr="0xf0000" size="0x10000" alias="0xff0000" file="ibm5150-bios.json5" />
    <video model="cga" memory="0x4000" screenWidth="640" screenHeight="400" />
    <fdc autoMount="{A:{name:'MS-DOS 6.22', path:'msdos622.img'}}" />
    <keyboard model="us84" />
</machine>

Сохраните этот XML как my-machine.xml. Затем создайте HTML-страницу, как в модуле 2, и загрузите её на сервер вместе с pcx86.js (скачанным с сайта PCjs). Убедитесь, что файлы BIOS и образ диска лежат по указанным путям.

⚠️ Где взять образы BIOS и дисков?
BIOS-файлы (в формате JSON5) можно скачать с сайта PCjs по прямым ссылкам (например, /machines/pcx86/ibm/5150/rom/bios/1982-04-24/BIOS.json5). Образы дискет – также с PCjs или создать самостоятельно с помощью утилит вроде WinImage. Но для учебных целей проще использовать готовые образы с сайта.

3.4. Тестирование и отладка

После того как XML готов, поместите его в одну папку с pcx86.js и HTML-файлом. Откройте HTML в браузере. Если эмулятор не запускается, откройте консоль разработчика (F12) и посмотрите на ошибки. Частые проблемы:

  • 404 Not Found – отсутствует файл, указанный в file или path. Проверьте пути.
  • Invalid XML – синтаксическая ошибка (не закрыт тег, пропущены кавычки). Валидируйте XML через онлайн-сервисы.
  • Нет изображения – возможно, неправильно указан screenWidth/Height или модель видео.
📝 Задание для самопроверки
1. Напишите XML-конфигурацию для IBM PC AT (80286) с 1 МБ RAM, EGA-видео и жёстким диском 20 МБ (используйте образ ibm5170-bios.json5 и hd20mb.img).
2. Подключите в качестве дисковода A образ дискеты с MS-DOS 3.30.
3. Загрузите эту конфигурацию в эмулятор и убедитесь, что система загружается.
4. Попробуйте изменить частоту процессора на 8 МГц и посмотреть, как изменится скорость эмуляции.
