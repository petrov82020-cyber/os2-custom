Модуль 6: Модификация эмулятора — добавление кнопок, управление через JavaScript, отладчик

✍️ Время чтения: ~25 минут | 🎯 Цель: научиться модифицировать эмулятор: добавлять новые элементы управления, управлять эмуляцией через JavaScript-код и использовать встроенный отладчик

На предыдущих этапах мы создали полноценную виртуальную машину, научились настраивать устройства и загружать операционные системы. Однако готовая конфигурация не всегда покрывает все нужды. В этом модуле мы рассмотрим, как модифицировать эмулятор, чтобы он лучше соответствовал вашим задачам: добавлять кнопки управления, управлять эмуляцией программно и отлаживать работу виртуальной машины.

💡 Ключевая идея
PCx86 предоставляет API для управления эмуляцией через JavaScript, позволяет расширять интерфейс с помощью XML-тегов `` и содержит мощный встроенный отладчик. Модификация эмулятора не требует изменения его исходного кода — всё делается через конфигурацию и вызовы функций.

6.1. Программное управление эмулятором

Функция embedPCx86() не только создаёт эмулятор, но и возвращает объект, с помощью которого можно управлять его работой. Этот объект содержит методы для запуска, остановки, сброса и доступа к компонентам.

6.1.1. Получение ссылки на объект эмулятора

let machine = embedPCx86("myMachine", "my-machine.xml", "components.xsl");

После этого вы можете вызывать методы:

machine.stop();      // остановить эмуляцию
machine.start();     // запустить/продолжить
machine.reset();     // аппаратный сброс
machine.powerDown(); // выключить питание

6.1.2. Управление состоянием (сохранение и загрузка)

PCx86 позволяет сохранять и загружать состояние эмулятора. Это полезно для создания «сейвов» или для возобновления работы с определённой точки. Состояние сохраняется в объекте, который можно сериализовать в JSON и восстановить позже.

let state = machine.saveState();
// ... сохранить state в localStorage или отправить на сервер ...
machine.loadState(state);

6.1.3. Доступ к компонентам (CPU, RAM, Video)

Через объект эмулятора можно получить доступ к отдельным компонентам, например, к процессору или видеопамяти:

let cpu = machine.getCPU();
console.log("Регистры CPU:", cpu.getRegisters());

let video = machine.getVideo();
video.screenshot(); // сделать скриншот экрана эмулятора

6.1.4. Пример: Кнопка «Сделать скриншот»

Вот как можно добавить на HTML-страницу кнопку, которая делает скриншот текущего экрана эмулятора:

<button id="screenshotBtn">Скриншот</button>
<script>
    let machine = embedPCx86("myMachine", "my-machine.xml", "components.xsl");
    document.getElementById("screenshotBtn").addEventListener("click", function() {
        let screenshot = machine.getVideo().screenshot();
        // screenshot — это DataURL изображения PNG
        let link = document.createElement("a");
        link.download = "screenshot.png";
        link.href = screenshot;
        link.click();
    });
</script>

6.2. Модификация XML-конфигурации: добавление кнопок и панелей управления

PCx86 позволяет добавлять кнопки управления прямо в XML-файл с помощью тега <control>. Эти кнопки автоматически появятся в интерфейсе эмулятора и будут выполнять предопределённые действия (запуск, остановка, сброс и т.д.).

6.2.1. Базовые типы элементов управления

  • <control type="button" binding="run"> — кнопка запуска эмуляции.
  • <control type="button" binding="stop"> — кнопка остановки.
  • <control type="button" binding="reset"> — кнопка аппаратного сброса.
  • <control type="button" binding="step"> — кнопка пошагового выполнения (работает только при включённом отладчике).
  • <control type="button" binding="ctrl-alt-del"> — отправка комбинации Ctrl+Alt+Del в эмулятор.

Пример панели с кнопками управления:

<panel>
    <control type="button" binding="run">Запустить</control>
    <control type="button" binding="stop">Остановить</control>
    <control type="button" binding="reset">Сброс</control>
    <control type="button" binding="step">Шаг</control>
    <control type="button" binding="ctrl-alt-del">Ctrl+Alt+Del</control>
</panel>

6.2.2. Добавление пользовательских кнопок

Вы можете создавать кнопки, которые вызывают произвольный JavaScript-код, используя атрибут onclick:

<panel>
    <control type="button" onclick="alert('Привет от эмулятора!')">Пользовательская кнопка</control>
</panel>

Этот код будет выполняться в контексте страницы, поэтому вы можете взаимодействовать с эмулятором через JavaScript, как было показано выше.

6.2.3. Создание собственной панели управления (пример)

Вот как можно добавить панель с кнопками для управления дисководами:

<panel>
    <control type="button" onclick="machine.getFDC().ejectDisk(0)">Извлечь дискету A</control>
    <control type="button" onclick="machine.getFDC().insertDisk(0, 'my-disk.json')">Вставить дискету A</control>
    <control type="button" onclick="machine.getHDC().saveDrive(0)">Сохранить жёсткий диск</control>
</panel>

6.3. Параметры функции embedPCx86()

Функция embedPCx86() принимает четыре параметра, три из которых обязательны, а четвёртый — опциональный объект с настройками. Полный список параметров и их значений описан в документации PCjs и в исходных кодах.

6.3.1. Обязательные параметры

  • id — идентификатор элемента <div>, в котором будет размещён эмулятор.
  • xml — путь к XML-файлу с конфигурацией машины.
  • xsl — путь к XSL-файлу (обычно components.xsl).

Пример вызова с обязательными параметрами:

embedPCx86("machineDiv", "config.xml", "components.xsl");

6.3.2. Четвёртый параметр (parms) — настройка эмулятора

Четвёртый параметр позволяет переопределить свойства машины, заданные в XML, а также управлять поведением эмулятора. Например, можно указать другой образ жёсткого диска или отключить автоматический запуск.

Структура объекта parms:

  • drives — массив объектов, описывающих жёсткие диски (переопределяет диски из XML).
  • autoMount — объект для автоматической загрузки дискет (переопределяет autoMount из FDC).
  • state — объект состояния эмулятора (если указан, эмулятор восстановит это состояние).
  • resume — имя файла для сохранения/восстановления состояния.

Пример вызова с переопределением жёсткого диска и дискеты:

embedPCx86("machineDiv", "config.xml", "components.xsl", {
    drives: [{ name: "20Mb HDD", type: 2, path: "my-hdd.json" }],
    autoMount: { A: { name: "MS-DOS 6.22", path: "msdos622.json" } }
});

Если нужно, чтобы эмулятор всегда запускался с чистого состояния (не восстанавливал предыдущее), можно передать {state: null}.

embedPCx86("machineDiv", "config.xml", "components.xsl", { state: null });

6.4. Использование встроенного отладчика PCx86

Отладчик — мощный инструмент для разработчиков. Он позволяет останавливать выполнение, просматривать содержимое памяти, регистров и устанавливать точки останова. Отладчик может быть полезен не только для отладки эмулятора, но и для анализа работы программ внутри эмулятора.

6.4.1. Включение отладчика

Чтобы включить отладчик, необходимо:

  1. Добавить в XML-файл компонент <debugger> (можно указать уровень сообщений: messages="fault" для вывода только ошибок).
  2. Добавить в панель управления элементы управления отладчиком: поле ввода команд, кнопки Run, Step, Reset.
  3. В HTML-странице загружать отладочную версию скрипта: pcx86-dbg.js вместо pcx86.js.

Пример XML-фрагмента с отладчиком:

<debugger id="debugger" messages="fault"></debugger>
<panel>
    <control type="text" binding="debugInput" width="400px"/>
    <control type="button" binding="debugEnter">Ввести команду</control>
    <control type="button" binding="run">Run</control>
    <control type="button" binding="step">Step</control>
    <control type="button" binding="reset">Reset</control>
</panel>

6.4.2. Команды отладчика

Отладчик PCx86 поддерживает команды, аналогичные классическому отладчику DEBUG из MS-DOS. Вот некоторые из них:

  • d [address] — дамп памяти (отобразить содержимое).
  • u [address] — дизассемблировать код.
  • r — отобразить регистры процессора.
  • g [address] — продолжить выполнение (возможно, до указанного адреса).
  • bp [address] — установить точку останова.
  • t — выполнить одну инструкцию (трассировка).

Команды вводятся в поле ввода (debugInput), и после нажатия кнопки debugEnter они отправляются отладчику. Отладчик может также выводить сообщения в консоль браузера, если включён соответствующий уровень логирования.

6.4.3. Пример отладки загрузчика MS-DOS

Допустим, мы хотим остановить выполнение эмулятора, когда управление передаётся на загрузочный сектор дискеты (адрес 0x7C00).

  1. Запустите эмулятор.
  2. В поле ввода отладчика введите bp 0x7c00 и нажмите Enter.
  3. Нажмите кнопку Run. Эмулятор начнёт выполнение, но остановится, как только достигнет адреса 0x7C00.
  4. Введите u 0x7c00, чтобы увидеть дизассемблированный код загрузчика.
  5. Используйте t для пошагового выполнения и d 0x7c00 для просмотра данных загрузчика.

Такой подход позволяет анализировать работу программ на низком уровне и отлаживать собственные загрузчики или операционные системы.

6.5. Работа с LocalStorage для сохранения пользовательских настроек

PCx86 может сохранять изменения пользователя (например, модифицированные образы дискет) в localStorage браузера. Это позволяет сохранять настройки между сеансами.

Пример сохранения состояния эмулятора в localStorage:

function saveMachineState() {
    let state = machine.saveState();
    localStorage.setItem("myMachineState", JSON.stringify(state));
}

function loadMachineState() {
    let state = localStorage.getItem("myMachineState");
    if (state) {
        machine.loadState(JSON.parse(state));
    }
}

Эти функции можно привязать к кнопкам в интерфейсе, добавив в XML соответствующие элементы управления с атрибутом onclick.

6.6. Практические примеры модификации

Пример 1: Добавление кнопки «Сброс» в панель управления

<panel>
    <control type="button" binding="reset">Сброс</control>
</panel>

Пример 2: Создание кнопки для быстрой загрузки тестовой дискеты

<panel>
    <control type="button" onclick="machine.getFDC().insertDisk(0, 'test-disk.json')">Загрузить тестовую дискету</control>
</panel>

Пример 3: Отладка программы на ассемблере в OS/2

Предположим, вы пишете программу на ассемблере под OS/2 и хотите отладить её прямо в эмуляторе. Вы можете загрузить отладочную версию эмулятора (pcx86-dbg.js), установить точку останова на нужном адресе и просмотреть содержимое регистров после каждого шага. Это значительно упрощает отладку низкоуровневого кода.

📝 Задание для самопроверки
1. Добавьте на свою страницу с эмулятором кнопку, которая при нажатии останавливает эмуляцию.
2. Создайте панель управления с кнопками «Запустить», «Остановить» и «Сброс».
3. Реализуйте функцию сохранения состояния эмулятора в localStorage и кнопку для её вызова.
4. Включите отладчик в своей конфигурации и попробуйте установить точку останова на адресе 0x7C00 (загрузочный сектор).
5. Используя отладчик, проследите за выполнением первых инструкций загрузчика DOS.

6.7. Устранение неполадок

  • Кнопки не появляются — убедитесь, что вы добавили тег <panel> в XML-конфигурацию и он находится внутри <machine>.
  • Отладчик не отвечает — проверьте, что вы загружаете pcx86-dbg.js вместо pcx86.js, и что в XML присутствует компонент <debugger>.
  • Ошибка "embedPCx86 is not defined" — убедитесь, что скрипт загружен и переменная доступна. Если вы используете отладочную версию, имя функции может отличаться (обычно embedPCx86 сохраняется).
  • Состояние не сохраняется — проверьте, что объект состояния не содержит циклических ссылок (JSON.stringify может их не обрабатывать). Используйте метод saveState(), который возвращает сериализуемый объект.
