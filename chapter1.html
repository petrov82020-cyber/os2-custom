Глава 1: Регистрация на GitHub и создание репозитория

✍️ Время чтения: ~10 минут | 🎯 Цель: получить аккаунт GitHub и создать первый репозиторий

Добро пожаловать в первую главу нашего экспресс-курса! Здесь вы научитесь основам работы с GitHub – платформой, которая будет хранить код вашего сайта и бесплатно его публиковать. Не пугайтесь, всё очень просто.

💡 Что такое GitHub?
GitHub – это веб-сервис для хранения и совместной работы над кодом. Он работает с системой контроля версий Git. Для нас важно, что GitHub предоставляет бесплатный хостинг статических сайтов через функцию GitHub Pages.

1.1 Регистрация учётной записи

  1. Откройте сайт github.com.
  2. Нажмите на кнопку Sign up (регистрация).
  3. Введите:
    • Email (рабочий, на него придёт подтверждение).
    • Password (придумайте надёжный пароль).
    • Username (ваше уникальное имя – оно будет частью адреса вашего сайта, например ваше_имя.github.io).
  4. Пройдите проверку (капчу).
  5. Нажмите Create account.
  6. Подтвердите email – перейдите по ссылке из письма.
⚠️ Важно: Запомните или запишите ваш username и пароль. Имя пользователя изменить потом сложно, поэтому выбирайте что-то удобное и профессиональное (например, ivan-petrov, а не crazyCat2025).

1.2 Вход в аккаунт и знакомство с интерфейсом

После регистрации вы попадёте на главную страницу (Dashboard). Обратите внимание на верхнее меню:

  • Repositories – список ваших репозиториев (пока пусто).
  • New – зелёная кнопка для создания нового репозитория.
  • Profile – ваша публичная страница (по адресу github.com/ваше_имя).

Не бойтесь нажимать на разные вкладки – вы ничего не сломаете. GitHub очень дружелюбен к новичкам.

1.3 Создание первого репозитория

Репозиторий – это папка, в которой хранятся все файлы вашего сайта (HTML, CSS, картинки). Для нашего курса мы создадим репозиторий, который станет основой сайта.

  1. Нажмите на зелёную кнопку New или перейдите по ссылке github.com/new.
  2. В поле Repository name введите имя, например my-first-site.
    (Можно любое, но без пробелов и специальных символов, только латиница, цифры, дефис или подчёркивание).
  3. Оставьте переключатель Public – это бесплатно и необходимо для GitHub Pages (приватные репозитории с Pages требуют платной подписки).
  4. В разделе Initialize this repository with отметьте Add a README file – это создаст файл README.md, который будет отображаться на главной странице репозитория.
  5. Нажмите Create repository.
🎉 Поздравляю! Вы только что создали свой первый репозиторий. Теперь у вас есть место, куда мы будем загружать файлы сайта.

1.4 Что такое README и как его редактировать

Файл README.md – это описание вашего проекта. Он пишется в формате Markdown (лёгкий язык разметки). Прямо сейчас вы можете его изменить:

  1. В вашем репозитории нажмите на файл README.md.
  2. Нажмите на значок карандаша (Edit).
  3. Напишите, например: 
    # Мой первый сайт
Это тестовый проект для изучения GitHub Pages.
  4. Прокрутите вниз до кнопки Commit changes – это сохранит изменения. Оставляйте комментарий, например «Обновил README».

Позже мы научимся загружать настоящие HTML-файлы, а пока важно понять: репозиторий готов.

1.5 Включение GitHub Pages (простым способом)

Хотя полноценную настройку Pages мы пройдём во второй главе, уже сейчас можно убедиться, что ваш репозиторий умеет публиковать сайты.

  1. В вашем репозитории перейдите во вкладку Settings (шестерёнка в правом верхнем углу).
  2. В левом меню выберите Pages (раздел «Code and automation»).
  3. В разделе Branch выберите main (или master), оставьте папку / (root).
  4. Нажмите Save.
  5. Через минуту вы увидите зелёное сообщение: Your site is published at https://ваше_имя.github.io/my-first-site/.

Если вы перейдёте по этой ссылке, то пока увидите страницу 404 – это нормально, ведь у нас нет файла index.html. Во второй главе мы создадим его и загрузим.

📝 Задание для самопроверки
1. Зарегистрируйтесь на GitHub (если ещё не сделали).
2. Создайте публичный репозиторий с именем my-first-site и файлом README.
3. Отредактируйте README, добавьте пару строк о себе.
4. Включите GitHub Pages (выберите ветку main, корневую папку).
5. Запишите адрес вашего будущего сайта: https://ваше_имя.github.io/my-first-site/.

1.6 Часто задаваемые вопросы

  • Можно ли изменить имя репозитория позже? Да, в настройках репозитория (Settings → General → Repository name). Но тогда изменится и адрес сайта.
  • Что делать, если я забыл пароль? Используйте ссылку «Forgot password» на странице входа.
  • Почему мой сайт не открывается после включения Pages? Подождите 1-2 минуты и проверьте, есть ли в репозитории файл index.html. Пока его нет, будет 404 – это не страшно.
