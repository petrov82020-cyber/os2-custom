Модуль 1: Архитектура PCx86 – компоненты виртуальной машины

✍️ Время чтения: ~15 минут | 🎯 Цель: понять, из каких частей состоит эмулятор и как они взаимодействуют

Прежде чем создавать свою виртуальную машину, нужно разобраться, из каких «кирпичиков» она строится. В этом модуле мы рассмотрим основные компоненты PCx86, их роль в эмуляции и то, как они описываются в XML-конфигурации.

🏗️ Общая идея
Эмулятор PCx86 представляет собой набор JavaScript-классов, каждый из которых отвечает за определённое устройство реального компьютера: процессор, память, видеокарту, дисководы и т.д. Конфигурационный XML-файл связывает эти классы в единую работающую систему, задавая параметры (тактовую частоту, объём памяти, адреса ПЗУ и т.п.).

1. Центральный процессор (CPU)

Сердце любой эмулируемой машины. PCx86 поддерживает процессоры Intel 8088, 80186, 80286 и 80386. В XML процессор описывается тегом <cpu>:

<cpu id="cpu286" model="80286" cycles="8000000" autostart="true" />
  • model – тип процессора (определяет набор инструкций).
  • cycles – тактовая частота в герцах (влияет на скорость эмуляции).
  • autostart – запускать ли выполнение кода сразу после загрузки страницы.

Процессор эмулирует выполнение машинных инструкций, прерывания, защищённый режим (для 286/386).

2. Память (RAM и ROM)

Оперативная память задаётся тегом <ram>, ПЗУ (BIOS) – тегом <rom>. Адресное пространство 8086/80286 – 1 МБ (20 бит), но может быть расширено.

<ram id="ramLow" addr="0x00000" size="0xa0000" test="false" />
<ram id="ramExt" addr="0x100000" size="0x160000" />
<rom id="romBIOS" addr="0xf0000" size="0x10000" alias="0xff0000" file="ATBIOS-REV3.json5" />
  • addr – начальный физический адрес.
  • size – размер в байтах (шестнадцатеричный).
  • file – путь к JSON-файлу с образом ПЗУ.

3. Видеосистема (Video)

PCx86 эмулирует видеоадаптеры MDA, CGA, EGA, VGA. Видео описывается тегом <video>, который может включать вложенное ПЗУ видеокарты.

<video id="videoEGA" model="ega" memory="0x20000" screenWidth="1280" screenHeight="700">
    <rom id="romEGA" addr="0xc0000" size="0x4000" file="IBM-EGA.json5" notify="videoEGA" />
</video>
  • model – тип адаптера (mda, cga, ega, vga).
  • memory – объём видеопамяти.
  • screenWidth/Height – размер canvas в пикселях.
  • ВидеоПЗУ обычно отображается по адресу 0xC0000.

4. Контроллеры дисков (FDC и HDC)

Floppy Disk Controller (FDC) и Hard Disk Controller (HDC) управляют образами дискет и жёстких дисков. В XML они выглядят так:

<fdc ref="library.xml" autoMount="{A:{name:'Disk1'}, B:{name:'Disk2'}}" />
<hdc id="hdcAT" type="AT" drives="[{name:'20Mb HDD', type:2, path:'my-hd.img'}]" />
  • autoMount – какие образы автоматически вставлять в дисководы.
  • drives – массив подключённых жёстких дисков.
  • path – путь к JSON-образу (или .img).

5. Клавиатура и мышь

Ввод данных эмулируется через компоненты <keyboard> и <mouse>. Для клавиатуры можно указать раскладку, для мыши – последовательный порт (обычно COM1).

<keyboard ref="us84-softkeys.xml" model="us84" />
<mouse serial="com1" />

6. Чипсет и прочие компоненты

Чипсет (<chipset>) определяет системную логику (например, модель 5170 для IBM PC AT). Также можно добавить последовательные порты (<serial>), панель управления (<panel>) и отладчик (<debugger>).

CPU

Эмулирует инструкции, регистры, прерывания. Отвечает за скорость работы.

RAM / ROM

Хранит код и данные. ROM содержит BIOS, который инициализирует систему.

Video

Формирует изображение, обрабатывает видеорежимы, текстовые буферы.

FDC / HDC

Обеспечивают чтение и запись образов дискет и жёстких дисков.

Keyboard / Mouse

Перехватывают события браузера и передают их эмулируемой системе.

Chipset

Связывает все компоненты, управляет шинами и таймерами.

Как компоненты общаются между собой?

В эмуляторе используется событийно-ориентированная модель. Например, CPU при обращении к порту ввода-вывода генерирует событие, которое перехватывается соответствующим устройством (видео, FDC). Адреса памяти распределяются между RAM, ROM и видеопамятью. XML-конфигурация задаёт эти диапазоны адресов.

📝 Задание для самопроверки
1. Откройте файл my-machine.xml, который вы использовали ранее. Найдите в нём описание CPU, RAM, ROM, видео.
2. Выпишите атрибуты для каждого компонента и попробуйте объяснить, что они означают.
3. Как вы думаете, почему у <ram> указан test="false"? (Подсказка: это отключает тест памяти в BIOS, ускоряя загрузку.)
4. Какой адрес у видеопамяти EGA? (Обычно 0xB8000 для текстового режима или 0xA0000 для графического.)
